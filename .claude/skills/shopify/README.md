# Shopify Skill

`/ck:shopify` helps users build Shopify apps, extensions, themes, billing flows, webhooks, and Liquid storefront work with current Shopify CLI and GraphQL-first guidance.

## File Map

- `SKILL.md` - user-facing routing, quick start, and high-level decision guide.
- `references/app-development.md` - OAuth, GraphQL Admin API, app config, webhooks, billing, metafields, and rate limits.
- `references/extensions.md` - Checkout/Admin/POS/Customer Account UI extensions and Shopify Functions.
- `references/themes.md` - Liquid theme syntax, sections, snippets, workflows, and theme patterns.
- `scripts/shopify_init.py` - thin wrapper around official Shopify CLI commands.
- `scripts/tests/test_shopify_init.py` - unit tests for the wrapper; no real Shopify CLI or network required.

## Maintenance Workflow

1. Re-check official Shopify docs before changing examples.
2. Prefer GraphQL Admin API for new app guidance; keep REST wording legacy/migration-only.
3. Use `{api_version}` in reusable API snippets. Use concrete versions only for TOML/config examples that need them.
4. Keep `SKILL.md` concise; move detailed examples to `references/`.
5. Do not hand-write Shopify app or extension scaffolds in helper scripts; delegate to Shopify CLI.
6. Avoid examples that log OAuth codes, client secrets, access tokens, callback query strings, or customer/order webhook payloads.

## Quarterly Review Checklist

- Latest stable Admin API version and deprecations.
- GraphQL mutation signatures for products, billing, metafields, and webhooks.
- Shopify CLI app config/deploy semantics.
- Extension TOML structure generated by current CLI templates.
- Functions target-specific input/output shapes.
- Liquid image filters and theme best practices.
